## ⚠️ Проблемы и улучшения

### 🔴 Критичные проблемы

#### 1. Отсутствие обработки ошибок
**Проблема:** Нет отображения ошибок при загрузке или подтверждении сообщений.

**Текущий код:**
```typescript
const fetchInbox = useCallback(async () => {
  if (!agentId) return;
  try {
    const res = await fetch(`/api/messages/inbox?agentId=${agentId}`);
    // ...
  } catch {
    // Silent fail for polling ❌
  }
}, [agentId]);
```

**Рекомендация:**
- Добавить состояние `error` в `useInbox` hook
- Показывать toast/alert при ошибках
- Различать ошибки загрузки и ошибки подтверждения

#### 2. Нет оптимистичного UI при подтверждении
**Проблема:** Сообщение исчезает только после обновления через polling (до 3 секунд).

**Текущее поведение:**
1. Пользователь нажимает "Acknowledge"
2. Показывается spinner
3. Сообщение остаётся видимым
4. Через 3 секунды (polling) сообщение исчезает

**Рекомендация:**
- Удалять сообщение из списка сразу после успешного `ackMessage`
- Показывать анимацию исчезновения
- При ошибке — возвращать сообщение обратно

#### 3. Нет клавиатурной навигации
**Проблема:** Клавиатурные сокращения `j/k` и `a` для подтверждения не работают в inbox.

**Текущее состояние:**
- В `use-keyboard-shortcuts.ts` есть `ackMessage` handler
- В `InboxViewer` он не используется
- Нет выделения выбранного сообщения

**Рекомендация:**
- Добавить состояние `selectedIndex` для навигации
- Подключить `useKeyboardShortcuts` в `InboxViewer`
- Выделять выбранное сообщение визуально
- `a` — подтвердить выбранное сообщение
- `j/k` или `↑/↓` — навигация по сообщениям

### 🟡 Средние проблемы

#### 4. Нет группировки сообщений
**Проблема:** Все сообщения показываются плоским списком, нет группировки по:
- Отправителю
- Треду
- Времени (сегодня, вчера, на этой неделе)

**Рекомендация:**
- Группировать по треду (если `threadId` доступен)
- Показывать заголовок треда над группой
- Возможность развернуть/свернуть группу

#### 5. Нет информации о треде
**Проблема:** Не видно, к какому треду относится сообщение.

**Текущий код:**
```typescript
// В сообщении есть threadId, но он не отображается
```

**Рекомендация:**
- Показывать название треда в footer сообщения
- Добавить ссылку для перехода к треду
- Badge с ID треда (сокращённым)

#### 6. Нет фильтрации и поиска
**Проблема:** При большом количестве сообщений сложно найти нужное.

**Рекомендация:**
- Добавить поиск по тексту сообщения
- Фильтр по отправителю
- Фильтр по треду
- Сортировка (по дате, по отправителю)

#### 7. Нет индикации broadcast сообщений
**Проблема:** Не видно, что сообщение — broadcast (для всех агентов).

**Рекомендация:**
- Badge "Broadcast" для сообщений с `toAgentId = null`
- Иконка глобуса или мегафона
- Визуальное отличие (например, другой border color)

### 🟢 Мелкие улучшения

#### 8. Улучшение отображения времени
**Текущее:** `toLocaleString` с полной датой

**Рекомендация:**
- Относительное время: "2 минуты назад", "вчера"
- Абсолютное время при hover
- Timezone-aware форматирование

#### 9. Улучшение отображения payload
**Текущее:** `JSON.stringify` для сложных объектов

**Рекомендация:**
- Красивое форматирование JSON
- Поддержка разных типов payload (text, code, markdown)
- Collapsible для больших payload

#### 10. Улучшение индикатора polling
**Текущее:** Статичный текст "Auto-refreshing every 3 seconds"

**Рекомендация:**
- Показывать время до следующего обновления
- Индикатор последнего успешного обновления
- Возможность отключить auto-refresh

## 🔧 Технические детали

### Изменения в `use-messages.ts`

```typescript
export function useInbox(agentId: string | null, pollInterval = 3000) {
  const [messages, setMessages] = useState<Message[]>([]);
  const [loading, setLoading] = useState(false);
  const [error, setError] = useState<string | null>(null); // ✅ Добавить
  
  const ackMessage = useCallback(async (messageId: string) => {
    // ✅ Optimistic update
    setMessages((prev) => prev.filter((m) => m.id !== messageId));
    
    try {
      const res = await fetch(`/api/messages/${messageId}/ack`, {
        method: "POST",
      });
      if (!res.ok) {
        // ✅ Revert on error
        await fetchInbox();
        throw new Error("Failed to ack message");
      }
    } catch (err) {
      // ✅ Revert on error
      await fetchInbox();
      throw err;
    }
  }, [fetchInbox]);
}
```

### Изменения в `inbox-viewer.tsx`

```typescript
export function InboxViewer({ agentId }: Props) {
  const { messages, loading, error, ackMessage, refetch } = useInbox(agentId);
  const [selectedIndex, setSelectedIndex] = useState<number | null>(null);
  
  // ✅ Keyboard shortcuts
  useKeyboardShortcuts({
    selectNext: () => {
      setSelectedIndex((prev) => 
        prev === null ? 0 : Math.min(prev + 1, messages.length - 1)
      );
    },
    selectPrevious: () => {
      setSelectedIndex((prev) => 
        prev === null ? messages.length - 1 : Math.max(prev - 1, 0)
      );
    },
    ackMessage: () => {
      if (selectedIndex !== null && messages[selectedIndex]) {
        handleAck(messages[selectedIndex].id);
      }
    },
  });
  
  // ✅ Error state
  if (error) {
    return (
      <div className="h-full rounded-lg border bg-card flex flex-col items-center justify-center">
        <AlertCircleIcon className="size-12 mb-4 text-destructive" />
        <p className="font-medium text-destructive">Ошибка загрузки</p>
        <p className="text-sm mt-1 text-muted-foreground">{error}</p>
        <Button onClick={refetch} className="mt-4">Повторить</Button>
      </div>
    );
  }
}
```
