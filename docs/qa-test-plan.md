## 4. API тестовые сценарии

### 🔹 4.1. Happy Path — Полный цикл

**Цель:** Проверить полный цикл отправки и получения сообщения

| Шаг | Действие | Ожидаемый результат | Проверка |
|-----|----------|---------------------|----------|
| 1 | Создать Agent A | 201, `id` возвращен | ✅ |
| 2 | Создать Agent B | 201, `id` возвращен | ⬜ |
| 3 | Создать Thread | 201, `id` возвращен | ⬜ |
| 4 | Отправить сообщение A→B | 201, `status: "pending"` | ⬜ |
| 5 | Получить inbox Agent B | 200, сообщение, `status: "delivered"` | ⬜ |
| 6 | Повторный inbox запрос | 200, `status: "delivered"` (идемпотент) | ⬜ |
| 7 | ACK сообщение | 200, `status: "ack"` | ⬜ |
| 8 | Получить inbox Agent B | 200, пустой массив | ⬜ |

**cURL команды:**

```bash
# 1. Создать агентов
AGENT_A=$(curl -s -X POST http://localhost:3333/api/agents \
  -H 'Content-Type: application/json' \
  -d '{"name":"AgentA","role":"dev"}')

AGENT_B=$(curl -s -X POST http://localhost:3333/api/agents \
  -H 'Content-Type: application/json' \
  -d '{"name":"AgentB","role":"qa"}')

# 2. Создать тред
THREAD=$(curl -s -X POST http://localhost:3333/api/threads \
  -H 'Content-Type: application/json' \
  -d '{"title":"Test Thread"}')

# 3. Отправить сообщение
MESSAGE=$(curl -s -X POST http://localhost:3333/api/messages/send \
  -H 'Content-Type: application/json' \
  -d "{\"threadId\":\"$THREAD_ID\",\"fromAgentId\":\"$AGENT_A_ID\",\"toAgentId\":\"$AGENT_B_ID\",\"payload\":{\"text\":\"Hello\"}}")

# 4. Получить inbox
INBOX=$(curl -s "http://localhost:3333/api/messages/inbox?agentId=$AGENT_B_ID")

# 5. ACK сообщение
ACK=$(curl -s -X POST "http://localhost:3333/api/messages/$MESSAGE_ID/ack")
```

## 6. DLQ и Retry логика

### 🔹 6.1. Retry механизм

**Параметры:**
- `MAX_RETRIES = 3`
- `DELIVERY_TIMEOUT_MS = 60_000` (1 минута)

| № | Тест | Ожидаемый результат | Статус |
|---|------|---------------------|--------|
| R1 | Сообщение в статусе `delivered` > 1 мин | Переходит в `pending`, `retries++` | ⬜ |
| R2 | После 3 повторов → `dlq` | `status: "dlq"`, `retries: 3` | ⬜ |
| R3 | Retry worker запускается периодически | Обрабатывает таймауты | ⬜ |
| R4 | `retries` увеличивается при каждом повторе | Счетчик корректен | ⬜ |

**Тестовый сценарий:**

```bash
# 1. Отправить сообщение
# 2. Получить inbox (pending → delivered)
# 3. НЕ делать ACK
# 4. Подождать > 1 минуты
# 5. Запустить retry worker: pnpm worker:retry
# 6. Проверить статус: delivered → pending, retries++
# 7. Повторить 3 раза
# 8. Проверить: status = dlq, retries = 3
```
