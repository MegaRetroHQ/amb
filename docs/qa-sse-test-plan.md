### ✅ 3. Connection Limits (6 на домен)

**Проблема:**
- Браузеры ограничивают количество одновременных HTTP соединений к одному домену
- Обычно лимит: 6 соединений для HTTP/1.1
- SSE использует одно соединение, но нужно проверить

**Тесты:**
- [ ] Проверить количество одновременных SSE соединений
- [ ] Убедиться, что используется только 1 SSE stream
- [ ] Проверить, что нет конфликтов с другими запросами
- [ ] Проверить работу при открытии нескольких вкладок

**Ожидаемый результат:**
- ✅ Только 1 SSE соединение к `/api/stream`
- ✅ Все компоненты используют один и тот же stream
- ✅ Нет превышения лимита соединений

---

### ✅ 4. Производительность vs Polling

**Текущая реализация:**
- SSE: обновления каждые 3 секунды через один stream
- Polling (старый): каждый компонент делает отдельный запрос каждые 3 секунды

**Метрики для сравнения:**
- [ ] Количество HTTP запросов в секунду
- [ ] Использование памяти
- [ ] Задержка обновления данных
- [ ] Нагрузка на сервер

**Ожидаемые улучшения:**
- ✅ Меньше HTTP запросов (1 stream vs N polling запросов)
- ✅ Меньше нагрузка на сервер
- ✅ Более эффективное использование соединений
