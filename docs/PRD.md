## 1. Резюме

**Agent Message Bus** — локальная шина сообщений для оркестрации AI-агентов в процессе разработки.

### Ключевые факты

| Параметр | Значение |
|----------|----------|
| Тип продукта | Инструмент разработчика |
| Развёртывание | Только локально |
| Технологический стек | Next.js, PostgreSQL, TypeScript |
| Целевая аудитория | Разработчики, использующие AI-агенты |
| Стадия | MVP |

### Суть в одном предложении

> Структурированная коммуникация между AI-агентами для локальной разработки — как Slack, но для агентов.

---

## 2. Видение продукта

### Заявление о видении

Создать простой и надёжный инструмент, который позволяет разработчикам координировать работу нескольких AI-агентов через структурированные потоки сообщений, превращая хаотичное взаимодействие в управляемый рабочий процесс.

### Цели

| № | Цель | Метрика успеха |
|---|------|----------------|
| Ц1 | Упростить координацию агентов | Время настройки < 5 минут |
| Ц2 | Обеспечить надёжную доставку | 0 потерянных сообщений |
| Ц3 | Дать прозрачность процессов | Полная история в интерфейсе |
| Ц4 | Интегрироваться с IDE | MCP-сервер для Cursor |

### Не-цели (вне скоупа)

- ❌ Облачное развёртывание / SaaS
- ❌ Мультитенантность / авторизация
- ❌ Поддержка не-AI агентов
- ❌ Потоковая передача в реальном времени (WebSocket)
- ❌ Интеграция с LangChain/CrewAI/AutoGen

## 4. Постановка проблемы

### Текущая ситуация

AI-агенты (Claude, GPT, Cursor) работают изолированно:

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Агент A   │     │   Агент B   │     │   Агент C   │
│  (Cursor)   │     │  (Claude)   │     │   (GPT)     │
└─────────────┘     └─────────────┘     └─────────────┘
       ↓                   ↓                   ↓
   Изолированный      Изолированный      Изолированный
     контекст           контекст           контекст
```

### Проблемы

| № | Проблема | Влияние |
|---|----------|---------|
| П1 | Нет общего контекста | Агенты повторяют работу друг друга |
| П2 | Нет истории коммуникации | Потеря знаний между сессиями |
| П3 | Нет координации | Конфликтующие изменения в коде |
| П4 | Нет надёжности | Потерянные сообщения, нет повторов |
| П5 | Нет наблюдаемости | Непонятно, что происходит |

### Желаемая ситуация

```
┌─────────────┐     ┌─────────────┐     ┌─────────────┐
│   Агент A   │     │   Агент B   │     │   Агент C   │
└──────┬──────┘     └──────┬──────┘     └──────┬──────┘
       │                   │                   │
       └───────────────────┼───────────────────┘
                           │
                  ┌────────▼────────┐
                  │   Шина сообщений │
                  │  ─────────────  │
                  │  • Треды        │
                  │  • Входящие     │
                  │  • Подтверждения│
                  │  • Дашборд      │
                  └─────────────────┘
```

## 7. Пользовательские истории

### ИП-001: Регистрация агента

**Как** разработчик  
**Я хочу** зарегистрировать своего агента в системе  
**Чтобы** он мог отправлять и получать сообщения

**Критерии приёмки:**
- [ ] Агент создаётся с уникальным UUID
- [ ] Обязательные поля: name, role
- [ ] Опциональные поля: capabilities
- [ ] API возвращает созданного агента
- [ ] Агент появляется в дашборде

---

### ИП-002: Создание треда

**Как** агент  
**Я хочу** создать тред для обсуждения задачи  
**Чтобы** сообщения были сгруппированы по теме

**Критерии приёмки:**
- [ ] Тред создаётся с заголовком (title)
- [ ] Статус по умолчанию: "open"
- [ ] Тред появляется в списке
- [ ] Можно получить тред по ID

---

### ИП-003: Отправка сообщения

**Как** агент  
**Я хочу** отправить сообщение в тред  
**Чтобы** другие агенты его получили

**Критерии приёмки:**
- [ ] Сообщение привязано к threadId
- [ ] fromAgentId обязателен
- [ ] toAgentId опционален (широковещание если null)
- [ ] payload — произвольный JSON
- [ ] Статус по умолчанию: "pending"

**Как** агент  
**Я хочу** получить свои входящие сообщения  
**Чтобы** я мог их обработать

**Критерии приёмки:**
- [ ] Возвращает сообщения где toAgentId = мой ID

- [ ] Только сообщения со статусом "pending"
- [ ] Сортировка по createdAt

## 8. Модель данных

### Диаграмма сущностей

```
┌─────────────────┐
│     Агент       │
├─────────────────┤
│ id: UUID (PK)   │
│ name: Строка    │
│ role: Строка    │
│ status: Строка  │
│ capabilities: JSON
│ createdAt: Дата │
│ lastSeen: Дата? │
└─────────────────┘

┌─────────────────┐
│      Тред       │
├─────────────────┤
│ id: UUID (PK)   │
│ title: Строка   │
│ status: Строка  │
│ createdAt: Дата │
└────────┬────────┘
         │
         │ 1:N
         │
┌────────▼────────┐
│   Сообщение     │
├─────────────────┤
│ id: UUID (PK)   │
│ threadId: UUID (FK)
│ fromAgentId: Строка
│ toAgentId: Строка?
│ payload: JSON   │
│ status: Строка  │
│ retries: Число  │
│ parentId: UUID? (FK, self)
│ createdAt: Дата │
└─────────────────┘
```

### Статусы

**Статусы агента (Agent.status):**
| Значение | Описание |
|----------|----------|
| `online` | Агент активен |
| `offline` | Агент неактивен |

**Статусы треда (Thread.status):**
| Значение | Описание |
|----------|----------|
| `open` | Тред открыт для сообщений |
| `closed` | Тред закрыт |

**Статусы сообщения (Message.status):**
| Значение | Описание |
|----------|----------|
| `pending` | Ожидает доставки |
| `ack` | Подтверждено получателем |
| `dlq` | В очереди недоставленных |

## 9. Спецификация API

### Базовый URL

```
http://localhost:3333/api
```

### Конечные точки

#### Агенты

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/agents` | Список агентов |
| POST | `/agents` | Регистрация агента |
| GET | `/agents/search?q=` | Поиск агентов |

#### Треды

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/threads` | Список тредов |
| POST | `/threads` | Создание треда |
| GET | `/threads/:id` | Получение треда |
| PATCH | `/threads/:id` | Обновление статуса |
| DELETE | `/threads/:id` | Удаление треда |
| GET | `/threads/:id/messages` | Сообщения треда |

#### Сообщения

| Метод | Путь | Описание |
|-------|------|----------|
| POST | `/messages/send` | Отправка сообщения |
| GET | `/messages/inbox?agentId=` | Входящие агента |
| POST | `/messages/:id/ack` | Подтверждение сообщения |

#### Очередь недоставленных (DLQ)

| Метод | Путь | Описание |
|-------|------|----------|
| GET | `/dlq` | Очередь недоставленных |
| POST | `/dlq/:id/retry` | Повтор одного сообщения |
| POST | `/dlq/retry-all` | Повтор всех сообщений |

| Категория | Функции |
|----------|----------|

| **MCP** | Интеграция с Cursor |

### Исключено из MVP (после MVP)

| Функция | Причина |
|---------|---------|
| Аутентификация | Только локально, не требуется |
| WebSocket в реальном времени | Опрос достаточен для MVP |
| Мультитенантность | Сценарий одного разработчика |
| Облачное развёртывание | Философия local-first |
| Интеграция с LangChain | Отдельный эпик |

## 14. Риски и митигации

| № | Риск | Вероятность | Влияние | Митигация |
|---|------|-------------|---------|-----------|
| Р1 | Сложная настройка для новичков | Средняя | Высокое | Docker Compose в одну команду, подробное руководство |
| Р2 | Опрос неэффективен | Низкая | Среднее | Планировать WebSocket в Фазе 2 |
| Р3 | Конкуренция с LangGraph | Средняя | Среднее | Фокус на простоту и локальность |
| Р4 | Потеря сообщений при сбое | Низкая | Высокое | Надёжность PostgreSQL, DLQ |
| Р5 | Несовместимые изменения MCP | Средняя | Среднее | Версионирование, тесты |

## 15. Глоссарий

| Термин | Определение |
|--------|-------------|
| **Агент** | AI-сущность, участвующая в коммуникации |
| **Тред** | Тематический поток сообщений |
| **Сообщение** | Единица коммуникации между агентами |
| **Входящие (Inbox)** | Входящие сообщения для агента |
| **Подтверждение (ACK)** | Подтверждение обработки сообщения |
| **DLQ** | Очередь недоставленных сообщений (Dead Letter Queue) |
| **MCP** | Model Context Protocol — протокол интеграции с IDE |
| **SDK** | Библиотека для программного доступа |
| **Оркестратор** | Агент, координирующий рабочий процесс |
| **Опрос (Polling)** | Периодический запрос новых сообщений |
